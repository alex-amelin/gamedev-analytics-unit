## Зафиксированные решения PRD (фиксируются по ходу)
1. **Уточнение против аддендума брифа** (2026-05-23, из чтения кода directaiq). `table_metadata` в directaiq существует в схеме (lazy-init, `BaseScript._log_table_operation()`), НО загрузчик логов `p81_load_logs.py` его для чекпойнта НЕ использует — состояние «что загружено» определяется прямым `SELECT DISTINCT date`. Сверка строк подтверждена как **WARNING** (`p81_load_logs.py:763–786`), не fail. CAST подтверждён строгим (HUGEINT, падает день при битой ячейке). Усиливает наши осознанные отличия: настоящая мета-таблица состояния + жёсткий fail + TRY_CAST.
2. **Подсказка по hot-window из directaiq** (2026-05-23). directaiq применяет 21-дневное hot-window для атрибуционных данных Метрики (`t10`), но это про доезжающую атрибуцию, не про сырые визиты/хиты. Для нашего сырья доезд короче → `[ДОПУЩЕНИЕ] N=3 дня`, уточнить на ревью (Open Question #2).
